API Dokumentation

Byg kraftfulde integrationer med Roasters REST API. Synkroniser vagtplaner, medarbejderdata og compliance-rapporter med dine eksisterende systemer.

Hurtig start

  • Autentificering

Autentificering

Alle API-anmodninger skal autentificeres med en API-nøgle. Du finder din nøgle i dashboardet under Indstillinger → API.

# Eksempel på anmodning

curl -X GET "https://api.roaster.dk/v1/shifts" \

-H "Authorization: Bearer YOUR_API_KEY" \

-H "Content-Type: application/json"

Endpoints

GET /v1/shifts

Hent alle vagter for din organisation. Understøtter filtrering på dato, medarbejder og afdeling.

POST /v1/shifts

Opret en ny vagt. Compliance-validering udføres automatisk.

GET /v1/employees

Hent liste over medarbejdere med deres tilgængelighed og kompetencer.

GET /v1/compliance/report

Generer en Arbejdsmiljøloven compliance-rapport for en given periode.

Rate limits

Standard rate limit er 1.000 anmodninger per minut. Enterprise-kunder kan få forhøjet limit ved behov. Kontakt support for mere information.

Webhooks

Konfigurer webhooks for at modtage realtidsopdateringer om vagtændringer, fravær og compliance-advarsler.

shift.created

Når en ny vagt oprettes

shift.updated

Når en vagt ændres

absence.requested

Når en fraværsanmodning indsendes

compliance.warning

Når en compliance-advarsel udløses
