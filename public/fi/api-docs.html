RESTful API

The Roaster API is organized around REST. Our API has predictable resource-oriented URLs, accepts JSON-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

Base URL: https://api.roaster.no/v1
Authentication: OAuth 2.0 / API Keys
Rate Limits: 1000 requests/minute (Pro), 5000 requests/minute (Enterprise)
Format: JSON request/response
Authentication 
# Using API Key
curl -X GET https://api.roaster.no/v1/rosters \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"

# Response
{
  "data": [
    {
      "id": "rst_123abc",
      "name": "Week 3 Schedule",
      "status": "published",
      "startDate": "2025-01-13",
      "endDate": "2025-01-19"
    }
  ],
  "meta": {
    "total": 42,
    "page": 1
  }
}

Quick Start Guide

Get up and running in minutes

1

Get Your API Key

Generate an API key from your Roaster dashboard under Settings > API Access. Keep your key secure and never expose it in client-side code.

Go to API Settings
2

Install the SDK

Use our official SDKs for Node.js, Python, or PHP to simplify integration.

npm install @roaster/sdk
View all SDKs
3

Make Your First Call

Test the connection by fetching your rosters. If successful, you're ready to build!

GET /v1/rosters
API Reference

Core Endpoints

The main resources available through the API

Rosters

Create, read, update, and publish work schedules.

GET /rosters
POST /rosters
PATCH /rosters/:id

Shifts

Manage individual shift assignments with compliance validation.

GET /shifts
POST /rosters/:id/shifts
DEL /shifts/:id

Employees

Access employee profiles, preferences, and hours.

GET /employees
GET /employees/:id/shifts
GET /employees/:id/hours

Time Off

Handle vacation, sick leave, and other time-off requests.

GET /timeoff
POST /timeoff
PATCH /timeoff/:id/approve

Compliance

Access compliance reports and validation results.

GET /compliance/report
GET /compliance/violations
POST /compliance/validate

Timesheets

Export worked hours for payroll integration.

GET /timesheets
POST /timesheets/export
GET /timesheets/summary

Webhooks

Get real-time notifications when events occur in Roaster. Subscribe to webhooks for schedule changes, approval requests, and more.

Available Events:

  • roster.published
  • shift.created
  • shift.updated
  • timeoff.requested
  • swap.approved
  • compliance.violation
