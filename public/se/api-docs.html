API Dokumentation

Bygg kraftfulla integrationer med Roasters REST API. Synkronisera scheman, anställdadata och efterlevnadsrapporter med dina befintliga system.

Autentisering

Alla API-anrop måste autentiseras med en API-nyckel. Du hittar din nyckel i dashboarden under Inställningar → API.

# Exempel på anrop

curl -X GET "https://api.roaster.se/v1/shifts" \

-H "Authorization: Bearer YOUR_API_KEY" \

-H "Content-Type: application/json"

Endpoints

GET /v1/shifts

Hämta alla skift för din organisation. Stöder filtrering på datum, anställd och avdelning.

POST /v1/shifts

Skapa ett nytt skift. Efterlevnadsvalidering utförs automatiskt.

GET /v1/employees

Hämta lista över anställda med deras tillgänglighet och kompetenser.

GET /v1/compliance/report

Generera en Arbetstidslagen-efterlevnadsrapport för en given period.

Rate limits

Standard rate limit är 1 000 anrop per minut. Enterprise-kunder kan få förhöjd limit vid behov. Kontakta support för mer information.

Webhooks

Konfigurera webhooks för att ta emot realtidsuppdateringar om skiftändringar, frånvaro och efterlevnadsvarningar.

shift.created

När ett nytt skift skapas

shift.updated

När ett skift ändras

absence.requested

När en frånvarobegäran skickas

compliance.warning

När en efterlevnadsvarning utlöses
